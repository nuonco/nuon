## At a glance

Nuon offers two primary methods for configuring and managing your app: the Nuon CLI and the Nuon Terraform provider or the Nuon API. This guide will concentrate on the former, as it is the quickest way to begin using Nuon.

In this quickstart guide, we will first install the Nuon CLI and Terraform providers. Next, we'll create an account and optionally link it to GitHub. After that, we'll configure your app, create installs using Terraform, and finally, release your app.

## Installing the CLI and Terraform provider
TKTK

## Create your account
```sh
nuon login
```

## Create an org

```sh
nuon orgs create --name --sandbox-mode
```

### (Optional) Connect a GitHub account
```sh
nuon orgs connect-github
```

## Configure your app using Terraform
tktk

### Creating your app
<CodeGroup>

```hcl outputs.tf
output "example" {
  value = module.example
}
```

```hcl inputs.tf
variable "example" {
  type        = string
  description = "example tf file"
}
```

```hcl data.tf
data "example" "current" {}
```

```hcl variables.tf
variable "example" {
  type        = string
  description = "example tf file"
}
```

</CodeGroup>

### Creating installs of your app

<CodeGroup>

```sh nuon cli
nuon installs create -n quickstart -r us-east-1 -o {aws-iam-role} --inputs eks_version=lastest
```

```hcl variables.tf
variable "install_id" {
  type        = string
  description = "install_id"
}

variable "region" {
  type        = string
  description = "region"
}

variable "public_domain" {
  type        = string
  description = "the root domain for the install. Usually {{.nuon.install.public_domain}}"
}

variable "public_domain_zone_id" {
  type        = string
  description = "the zone id for the public domain for the install"
}
```

</CodeGroup>

## Releasing your app
tktk

### Create a build
```sh
nuon builds create -c {component-name || component-id}
```

### Create a release
```sh
nuon releases create -c {component-name || component-id}
```
