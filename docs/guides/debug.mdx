## Debug

When incidents happen, you will want to respond in a timely manner, and ensure your customer remains confident you are prepared to handle issues for them.

### Key Questions

- What are the main issues you foresee having with your app in a customer's account? You should have actions to run common mitigations for these.
- How can you ensure no data is lost in the event you need to tear down and re-deploy components?

### Checklist

- runbooks
  - vendor maintenance runbook
  - break glass runbook
- actions
  - mitigations for common, known issues
