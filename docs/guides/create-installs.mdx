Nuon offers multiple ways to create installs, allowing to implement and manage this important process in the way that best suits your business needs.

<Note>
This guide assumes that you have create an IAM role, granting Nuon access to provision an AWS install. If not, please follow [this guide](/guides/install-access-permissions) first.
</Note>

## Installers

If you want to offer your customers a self-serve experience, you can create an installer.
Installers bundle together all the config required to present a friendly UX your customers can use to install your app.

Reference the [Installers guide](/guides/installers) to learn more.

## CLI

Installs can be created via the [Nuon CLI](../cli).

```sh
nuon installs create  --name=auto-deploy --region=us-east-1 --role=iam-role --inputs=eks_version=1.28
```

## Terraform

Installs can be created and managed using [Terraform](https://registry.terraform.io/providers/nuonco/nuon/latest/docs/resources/install).

Example of creating an install using Terraform:

```hcl
resource "nuon_install" "install" {
  app_id = nuon_app.<your-app>.id

  name         = "nuon-test-install"
  region       = "us-east-1"
  iam_role_arn = var.install_role_arn

  input {
    name = "aws-eks-version"
    value = "v.1.28"
  }
}
```

<Note>
Please refer to the [Terraform configuration management guide](./terraform-configuration-management.mdx) for more
details on working with the terraform provider.
</Note>

## API and SDKs

Installs can be created directly via our API, or by using an SDK:

* [API](https://docs.nuon.co/api-ref/installs/create-an-app-install)
* [SDKs](../sdks)

An example of using [nuon-go](https://github.com/nuonco/nuon-go) to create an install as part of a signup endpoint:
<CodeGroup>
```go installs.go
type CreateInstallRequest struct {
  Name string `json:"name" validate:"required"`

  AWSAccount struct {
    Region     string `json:"region"`
    IAMRoleARN string `json:"iam_role_arn" validate:"required"`
  } `json:"aws_account" validate:"required"`

  AppName string `json:"app_name" validate:"required"`
}

func (c *CreateInstallRequest) Validate(v *validator.Validate) error {
  if err := v.Struct(c); err != nil {
    return fmt.Errorf("invalid request: %w", err)
  }
  return nil
}

func (s *app) signup(ctx *gin.Context) {
  var req CreateInstallRequest
  if err := ctx.BindJSON(&req); err != nil {
    ctx.Error(fmt.Errorf("unable to parse request: %w", err))
    return
  }
  if err := req.Validate(s.v); err != nil {
    writeErr(ctx, err)
    return
  }

  app, err := s.apiClient.GetApp(ctx, req.AppName)
  if err != nil {
    writeErr(ctx, err)
    return
  }

  install, err := s.apiClient.CreateInstall(ctx, app.ID, &models.ServiceCreateInstallRequest{
    Name: &req.Name,
    AwsAccount: &models.ServiceCreateInstallRequestAwsAccount{
      IamRoleArn: &req.AWSAccount.IAMRoleARN,
      Region:     req.AWSAccount.Region,
    },
  })
  if err != nil {
    writeErr(ctx, err)
    return
  }

  ctx.JSON(http.StatusCreated, install)
}
```
</CodeGroup>
